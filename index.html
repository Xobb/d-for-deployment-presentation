nginx (on your VPS)

Зробіть додатковий віртуалхост.


            # /etc/nginx/sites-available/mywebsite.maintenance.conf
            location / {
            # Виключаємо для всіх окрім
            allow 1.2.3.4; # IP адресу, з якої ми будемо перевіряти чи все загарзд.
            deny all;
            error_page 403 maintenance.php;

            # далі йде ваш звичайний конфіг
            # ...
            }

            location /maintenance.php {allow all;}

Включити:


            sudo rm -rf /etc/nginx/sites-enables/mywebsite.conf
            sudo ln -s /etc/nginx/sites-available/mywebsite.maintenance.conf
            /etc/nginx/sites-enabled/mywebsite.maintenance.conf
            sudo service nginx configtest && sudo service nginx restart

Виключити:


            sudo rm -rf /etc/nginx/sites-enables/mywebsite.maintenance.conf
            sudo ln -s /etc/nginx/sites-available/mywebsite.conf /etc/nginx/sites-enabled/mywebsite.conf
            sudo service nginx configtest && sudo service nginx restart

Revolution!

Наслідки

  • Слово DevOps починається використовуватись в закутках софтварних компаній, ще не популярне воно зовсім.
  • З'являється можливість розвивати opensource проекти з максимальним залученням community і з кожним роком стає все більш доступно. Ще не з'явився github, але відправити патча, який в кінці змерджить сам Лінус стає можливим.
  • Багато бізнесів отримують можливість оперувати великою кількістю серверів не збільшуючи при цьому кількість адміністраторів. Знята ще одна перешкода для формування стартапської культури.
  • Коммуніті веб-розробників чи не вперше масово об'єднується в групи і розробляє флагманські фреймворки, також користуючись допомогою коммуніті зменшується кількість необхідних розробників для продуктів.
  • Ющенко починає просерати всі полімери =)

Новинки того року

  • Скриптові аплікейшони почали більдити. (Не php звісно, тим не менше). Тепер почали тестувати більди.
  • З'явились адекватні інструменти для автоматичного тестування, йому почали приділяти увагу. Однією складовою більд процесу став прогін юніт, функціональних та прийомних тестів.
  • Така автоматизація дозволила провіжинити з bare-metal до продакшон системи за лінійний час без участі системного адміністратора.
  • Так як це призвело до подешевшання VPS то проекти, що приносили гроші почали мігрувати з Shared-Hosting'а.
  • Так як ми почали деплоїти збілджені пакети, то emerge world почав пропадати з сервера, які і інші source-based дистрибутиви.

Підходи до деплою

  • Легко робиться maintenance сторінка під час деплою.
  • Можливий не тільки оновлення коду, але і зміна конфігурації.
  • Атомарність процедури деплою. Так би мовити транзакційність.
  • Автоматизований роллбек при помилці.
  • Перші кроки до zero-downtime deploy.

PaaS та IaaS

Платформи -- набір пре-конфігурованих серверів, що готові приймати ваш аплікейшон і одразу його запускати. Приклади: Heroku, CenturyLink, etc. Перевагою платформи є швидкість розгортання і відсутність необхідності і Ops`у. Мінусами ж є відсутність можливості кастомізації платформи, встановлення додаткового програмного забезпечення і ціна. Більше підходить для стартапів з невеликими навантаженнями на початковому рівні розвитку.

Інфраструктура в свою чергу являється серверами, віртуальними чи фізичними, мережею, сховищем інформації і іншими низькорівневими компонентами, які ви орендуєте на умовах власника. Найзнатнішим прикладом є AWS. Також відносять туди JoyentCloud, DigitalOcean, Linode, Slicehost, хоча останні три являються скоріше надають VPS на стероїдах.
